## What was tried
- Defined `inductive ProtTech | L | F | LF | Custom (bits : List Bool)`
- `tech_to_mask : ProtTech → List Bool` (maps to the existing L_mask etc.)
- `is_stable_for_tech (lam mu) (t : ProtTech) : Bool` delegating to the decider.
- This follows "creating custom inductive types for 'allowed grade sets'" and "structure on projectors + representation theory" strategies.
- Intended to make "only these discrete masks work" precise: L/F/LF are the named constructors corresponding to the Z2xZ2 from Hyp 1 (L=28, F=35, L+F=63).

## What worked
- Compiles cleanly; can now write `is_stable_for_tech 0.001 0.005 .LF` etc.
- Provides algebraic structure: pattern matching on which protection tech, easy to add lemmas like:
    `lemma LF_is_full : tech_to_mask .LF = Full_mask := rfl`
    (can be added; would be provable).
- Prepares for Phase 2 "7-fold degeneracy for quaternion-triple subalgebras" (L corresponds to a closed subalgebra in the Fano table).

## What failed / not yet

- The "prove mask properties" part of the angle is started but not deep (no `theorem` yet beyond the type).
- Still tied to the List representation internally.

## Current state

- Directly supports "make the 'only these discrete masks work' statement more precise" (roadmap Phase 2, but started in Phase 1).
- Nice for future: `match tech with | .L => ... | .F => ...` in theorems.
